# Как предложить доработку

Спасибо за интерес к проекту! Правки принимаются через **Pull Request**. Доступ на запись не нужен — достаточно форка.

## Быстрый путь

1. Нажмите **Fork** на странице репозитория — появится ваша копия.
2. Создайте ветку и внесите изменения:
   ```bash
   git clone https://github.com/<ваш-логин>/<репозиторий>.git
   cd <репозиторий>
   git checkout -b моя-правка
   # ... правки ...
   git commit -am "Коротко: что изменил"
   git push origin моя-правка
   ```
3. Откройте **Pull Request** в этот репозиторий с описанием, что и зачем меняете.
4. Мейнтейнер посмотрит, обсудит и смёржит.

Небольшую правку можно сделать прямо на GitHub: открыть файл → карандаш (Edit) → **Propose changes** → Create pull request.

## Где логика проекта

- Весь калькулятор — в одном файле **`index.html`** (HTML + CSS + JS, без бэкенда).
- Карта проекта и описание всех функций — в **[ARCHITECTURE.md](ARCHITECTURE.md)**. Начните с неё.
- Сердце расчётов — массив **`const scenarios = [`** в `index.html`: каждый объект = один блок калькулятора (поля, дефолты, формула `compute`).

Куда смотреть под задачу:
- изменить блок / формулу / поля → `const scenarios = [`;
- итоговый расчёт → `renderSummary` + `recompute`;
- PDF-отчёт → `buildReportHtml`;
- пояснения «i» → `const INFO`.

## Как проверить перед PR

- Откройте `index.html` двойным кликом в браузере и проверьте, что калькулятор считает и ничего не сломалось.
- Если правили формулы, дефолты сценариев, тарифную математику или настройку интегратора — прогоните тесты:
  ```bash
  npm test
  ```
- Если правили другой JavaScript, убедитесь, что в консоли браузера нет ошибок.

## Оформление

- Пишите в стиле окружающего кода (отступы, именование, язык комментариев — русский).
- Одна правка = один PR по возможности. Так быстрее ревью.
- Для визуальных изменений приложите скриншот в описание PR.

## Про две версии проекта

Проект живёт в двух репозиториях:
- **calc** — основная версия.
- **calc-serv** — параллельная (то же + блок «Настройка от интегратора»).

Предлагайте правку в тот репозиторий, которого она касается. Общие улучшения логики удобнее слать в **calc** — мейнтейнер перенесёт их в parallel-версию.
