# Калькулятор экономии BitrixGPT — версия для интеграторов

Партнёрская версия калькулятора экономического эффекта от внедрения **BitrixGPT** в Битрикс24.
Основа совпадает с основной версией [`BelMihMed/calc`](https://github.com/BelMihMed/calc), но добавлен блок **«Настройка от интегратора»** и контакты менеджера в отчёте.

## Возможности

- Сценарии, сгруппированные по CRM, ИИ-агентам, общим сценариям и VibeCode.
- Верхний блок «Данные о компании» автозаполняет связанные поля сценариев, но не перетирает ручные правки в сценариях.
- По каждому выбранному сценарию считается экономия ФОТ, потенциал/выручка и, в этой версии, разовая стоимость настройки интегратора.
- Интегратор может менять стоимость настройки по сценариям; значения сохраняются в `localStorage`.
- PDF-отчёт включает стоимость перехода, подписку BitrixGPT, настройку интегратора и блок контактов менеджера.
- Перед печатью отчёт открывается в полноэкранном предпросмотре; из него можно сохранить PDF или отправить документ на печать.
- В отчёте есть QR-код на ссылку менеджера (`qrLink`).
- Тарифы и подписки хранятся в едином `PRICE_CONFIG` с датой актуальности цен.
- Состав сформированных отчётов и клики по «Подробнее» могут логироваться в Google Sheets через Apps Script (`LOG_ENDPOINT`).

## Проверка формул

Тесты читают реальный `index.html`, извлекают из него сценарии и проверяют:
- базовые константы и дефолтные суммы сценариев;
- суммирование дополнительных сегментов;
- валидацию сценария поддержки;
- защиту ручных правок от автозаполнения;
- единый датированный конфиг тарифов;
- дефолтные стоимости настройки от интегратора.

```bash
npm test
```

На GitHub эти же тесты запускаются автоматически через GitHub Actions при `push` и `pull_request` в `main`.

## Тарифы и цены

Все тарифы, подписки BitrixGPT и коробочные подписки описаны в `PRICE_CONFIG` внутри `index.html`.
Там же хранится `actualOn` — дата актуальности цен, которая выводится в модалке выбора тарифа и в PDF-отчёте.

При обновлении цен меняйте только `PRICE_CONFIG` и запускайте `npm test`. Тест проверяет, что старые дублирующие словари цен не вернулись.

## Отчёты, контакты и аналитика

Отчёт формируется в браузере: HTML отчёта записывается в iframe предпросмотра, после чего пользователь сохраняет PDF через системную печать.

Перед формированием отчёта пользователь выбирает:
- формат внедрения: облако или коробка;
- текущий тариф/редакцию;
- название компании.

Контакты менеджера заполняются в блоке «Ваши контакты» и сохраняются в браузере через `localStorage`. В отчёт попадают имя, должность, компания, телефон, e-mail, Telegram и QR-ссылка, если они заполнены.

Если задан `LOG_ENDPOINT`, калькулятор отправляет в Apps Script события:
- `report` — состав отчёта, выбранный тариф, данные компании и рассчитанные суммы;
- `landing` — клик по ссылке «Подробнее».

Если `LOG_ENDPOINT` пустой, аналитика выключена.

## Отличия от основной версии

- Есть `INTEGRATOR_DEFAULT` — дефолтные разовые стоимости настройки по сценариям.
- `integratorCost(sc)` берёт значение из `localStorage` или из `INTEGRATOR_DEFAULT`.
- В итоговом блоке и PDF-отчёте стоимость настройки вычитается из выгоды первого года.
- В PDF-отчёте используется CTA с контактами менеджера вместо самоподключения.
